# XianYuPlus 更新日志

本文件记录各正式版本的主要变化。项目介绍与安装说明请查看 `README.md`。

## V1.9.2（2026-07-23）

### 自提订单回补

- 当闲鱼商家订单列表接口临时返回无权限时，手动同步会扫描本地近 30 天已保存的 WebSocket 交易卡片，补回带明确自提标记和订单 ID 的订单。
- WebSocket 实时交易新增识别 `onlyTakeSelf=true`；自提订单统一标记为 `PICKUP`，不会进入虚拟发货、手动发货或确认发货队列。
- 同步完成提示会显示从本地交易消息补回的自提订单数量，便于确认回补结果。

## V1.9.1（2026-07-23）

### 自提订单同步

- 修复已完成自提订单未进入订单管理的问题：兼容 `onlyTakeSelf=true` 标记、缺少商品 ID 的实时订单事件，以及缺少标准时间字段的最新订单列表记录。
- 自提订单会统一标记为 `PICKUP`，并移出虚拟发货、手动发货、确认发货等物流任务队列；已有记录再次同步后也会自动纠正。

### 账号管理响应式布局

- 账号管理改为按实际内容区宽度切换完整、紧凑和卡片布局，修复 13 英寸笔记本、分屏或浏览器缩放时账户卡片横向裁切的问题。

## V1.9.0（2026-07-22）

### 商品同步与稳定性

- 修复商品详情同步停留在 `0/1`、同步按钮持续转圈的问题；卡住或旧的详情任务会自动收口并释放账号同步状态。
- 修复异步详情同步的 Spring 代理类型错误。商品基础列表已保存时，即使详情补全未启动，也不会再误报账号连接异常。
- 商品售出下架后，“在售”分组为空会被视为正常同步完成；同步失败会展示具体账号与原因。

### 自提订单

- 自提订单会进入订单管理，并显示“自提待交接”。
- 自提订单不会进入自动发货队列，且禁止虚拟发货、手动发货和确认发货。
- 订单历史同步与实时交易消息都会识别自提标记，减少漏单与物流自动化误触发。

### WebSocket 与安全验证

- WebSocket Token 触发闲鱼网页安全验证后，会暂停该账号的自动重连，避免反复刷新 Cookie、重复请求 Token 与刷屏操作日志。
- 用户完成验证并更新凭证后再主动重连；闲鱼 App 不一定会弹出提示。

### 更新体验与构建

- 修复 WebSocket 安全验证提示的前端类型声明，恢复 Docker 前端生产构建。
- 更新弹窗优先显示 GitHub 最新正式 Release 的中文发布说明，不再展示开发提交标题；没有发布说明时才回退到提交摘要。

## V1.8.8（2026-07-22）

### WebSocket 安全验证

- WebSocket Token 请求触发闲鱼安全验证后，立即暂停该账号的自动重连，避免反复刷新 Cookie、重复请求 Token 和刷屏操作日志。
- 安全验证等待状态不再自动超时清除；只有用户完成验证后通过“凭证更新”重新连接时才会继续请求。
- 连接页面明确说明这是网页安全验证，闲鱼 App 不一定会弹出提示，并给出完成验证后的恢复步骤。

## V1.8.7（2026-07-22）

### 商品与订单同步

- 修复详情同步异步代理注入为具体实现类时触发的 Bean 类型错误；商品基础列表已同步后，不会再被误报为账号连接失败。
- 当闲鱼“在售”分组为空（例如商品已售出下架）时，商品同步会正确完成并提示当前没有在售商品。
- 同步失败提示改为展示具体账号与后端返回原因，便于区分账号会话问题与业务同步问题。

### 自提订单

- 自提订单会写入订单管理，并标记为“自提待交接”。
- 自提订单不会进入自动发货队列，也不允许虚拟发货、手动发货或确认发货。
- 订单历史同步与实时交易消息均会识别自提标识，避免漏单或误触发物流自动化。

## V1.8.6（2026-07-22）

### 商品同步

- 修复商品列表执行“同步闲鱼商品”后，详情同步停留在 `0/1` 导致按钮一直灰色转圈的问题。
- 后端详情补全任务增加超时收口机制，卡住或旧任务残留时会自动释放账号同步状态。
- 前端同步进度增加兜底超时判断，长时间无进度时自动结束等待并恢复按钮可点击。
- 商品基础列表同步成功后，即使详情补全受闲鱼接口或页面加载影响，也会保留已同步的基础商品信息。

## V1.8.5（2026-07-21）

### 自动化与交付

- AI 客服回复延迟改为系统级可配置项，可在“系统设置 → AI 客服配置”设置 1–60 秒，默认 15 秒，保存后即时生效。
- 商品卡密自动发货新增“自动确认发货”开关；卡密完整发送后才会执行确认发货。
- 自动评价在待评价列表暂未返回订单时，改由闲鱼最终评价接口核验，减少需要在执行中心手动“检查并评价”的情况。

### 备份与恢复

- 扩展手动备份：账号 Cookie 与自动化设置、全部系统设置、关键词规则、通知渠道、擦亮计划、黑名单、买家标签和商品素材库均可导出与恢复。
- 备份页面增加敏感信息提示，并明确导入采用按标识新增或更新的方式，不会删除备份中不存在的旧记录。
